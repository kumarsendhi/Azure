Lab Overview

In this lab, you learn the basics of authoring and deploying an Azure Resource Manager (ARM) template using Visual Studio 2017, focused on infrastructure as a service (IaaS) technologies. You will author a template that deploys a virtual machine, which gets automatically configured as a web server with a sample web app, as well as configuring a second virtual machine which gets configured with SQL Server and a database for the sample app.

Learning Objectives

·       Add the following resources to an ARM template:

o   Virtual Network

o   Virtual Machine

o   Storage Account

o   Public IP address

o   Network Security Group

o   PowerShell DSC extension

·       Add parameters and variables to an ARM template

·       Deploy the template to a resource group using Visual Studio 2017

Task 3: Create an Azure Storage Account

In this exercise, you will create a new storage account that will be used for deploying artifacts (PowerShell DSC Scripts) in future labs.

1.     From within the LABVM virtual machine, open your browser and navigate to https://portal.azure.com. 

 

2.     Enter the subscription associated with your Microsoft Azure subscription.

 

3.     If your account is associated with an organization account and a Microsoft account you may be prompted to choose which one to authenticate with for your Microsoft Azure account.

 

4.     Click New, Data + Storage, and then Storage Account.

      

 

5.     Change the deployment model to Resource Manager and click Create.

 

 

6.     Specify the following configuration and then click Create.

a)     Name: unique name for your storage account

b)    Deployment model: Resource Manager

c)     Account Kind: General Purpose

d)    Performance: Standard

e)     Replication: Locally-redundant storage (LRS)

f)      Storage service encryption: Disabled

g)    Subscription:  Select your subscription

h)     Resource Group: Create New / ARMTraining

i)       Location: The region closest to you
