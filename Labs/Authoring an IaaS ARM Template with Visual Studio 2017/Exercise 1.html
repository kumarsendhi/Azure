Exercise 1: Creating an ARM Template

Overview

In this lab, you will create a new Azure Resource Manager template add a virtual network and then deploy it to Microsoft Azure.

TIP:  In the Student Files for this lab there is a zip folder called FINAL-ARMIntro.zip which contains the completed Visual Studio solution.  This should only be used as a last resort troubleshooting step in case the deployment is not working and the student is unable to determine the syntax issues they are experiencing.  Two websites that are useful are http://www.jsonlint.com for determining if the JSON is valid and http://www.text-compare.com to see differences between your text files and the solution provided.

Time Estimate

30 minutes

Task 1: Create a new Azure Resource Manager Project

 

1.     Launch Visual Studio 2017 and Sign in using your Visual Studio Account. (Click Sign Up, if you dont have one and complete the process.)

 

2.     Within Visual Studio 2017 click File, New, and Project.

 

3.     Navigate to Templates, Visual C#, Cloud, and select Azure Resource Group.

 

4.     Change the name of the solution to: ARMLabIntro and the location to C:\OpsgilityTraining, and then click OK.

5.     Select the Blank Template, and click OK.

 

6.     Expand the ARMLabIntro project, and open the azuredeploy.json file.

 

 

Task 2: Add a Virtual Network to the ARM Template

1.     The azuredeploy.json file contains four different sections: parameters, variables, resources and outputs. On the left hand side, a new window called JSON Outline should have been opened as well. If this was not the case, go to the View menu->Other Windows and choose JSON Outline. The window should look like this:

 

2.     On the JSON Outline window press the Add Resource button on the top left corner, or right click on the resources and choose Add New Resource.

 

3.     On the Add Resource dialog choose Virtual Network and name it OPSTrainingVNET, then click Add.

 

4.     Go to the azuredeploy.json file, and inspect its content. Review the variables section. It should look like this:

 

    "variables": {

        "OPSTrainingVNETPrefix": "10.0.0.0/16",

        "OPSTrainingVNETSubnet1Name": "Subnet-1",

        "OPSTrainingVNETSubnet1Prefix": "10.0.0.0/24",

        "OPSTrainingVNETSubnet2Name": "Subnet-2",

        "OPSTrainingVNETSubnet2Prefix": "10.0.1.0/24"

 },

 

5.     Change the name of the Subnet-1 to FrontendNetwork, and the name of Subnet-2 to BackendNetwork. Save the file after making the changes.

 

    "variables": {

        "OPSTrainingVNETPrefix": "10.0.0.0/16",

        "OPSTrainingVNETSubnet1Name": "FrontendNetwork",

        "OPSTrainingVNETSubnet1Prefix": "10.0.0.0/24",

        "OPSTrainingVNETSubnet2Name": "BackendNetwork",

        "OPSTrainingVNETSubnet2Prefix": "10.0.1.0/24"

    },

6.     Deploy the template by right clicking on the project, and choosing Deploy->New Deployment.

 

7.     If you have not logged in to the account used for your Azure subscription yet, you will be asked to do so now.

 

8.     First fill in the email address associated with the Azure account and press continue.

 

9.     Select the Azure subscription you want to use for deploying the lab, and under the Resource Group dropdown click <Create New >.

 

 

10.  On the Create Resource Group dialog accept the default value for the name and choose the closest location to you, and press Create.

 

11.  When you are back on the Deploy to Resource Group dialog, press Deploy to execute the template.

 

12.  View the created resource group and virtual network in the Azure Management Portal by clicking the Resource Groups button and clicking the name of your resource group.

 

Summary

In this exercise, you started from creating an empty ARM template using Visual Studio 2017, added a virtual network with two subnets to it, and finally deployed the template to an Azure Resource Group.
