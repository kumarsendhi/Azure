Exercise 2: Creating and Configuring a SQL Database

Overview

In this exercise, you will create a SQL Database for the app, and configure connectivity using the SQL Database firewall rules.

Time Estimate

15 Minutes

Task 1: Create SQL Database

In this task, you will create SQL database, which will be used by a web application.

1.     While logged into labvm, log into the Azure portal (https://portal.azure.com).

 

2.     Click on New > Databases > SQL Database

 

3.     Specify the following configuration for the SQL Database.

·       Database name: eventsdb

·       Resource group: EventsDB_RG

·       Select source: Blank database

·       Server name: Create new -> (specify a unique name ex: entweb101)

·       Server admin login: demouser

·       Password: demo@pass123

·       Location: Location nearest you

·       Create V12 server (Latest update): Yes

·       Specify Standard S0 for the pricing tier of the SQL Database by selecting the Standard tab and moving the DTU Slider all the way to the left.  

 

 

4.     The deployment of the SQL Database takes ~5 minutes to complete.  You can monitor by clicking the notification icon on the top toolbar.

 

Task 2: Configure the SQL Server Firewall

In this task, you will configure SQL server firewall to accept connections from your local client machine.

1.     Click More Services > SQL Servers.

 

2.     Click on the server you have created in the previous exercise (entweb101).

 

3.     Click Firewall.

 

4.     Click on Add Client IP to add the IP address of the machine you are accessing the Azure portal from and the click Save.

 

Summary

In this exercise, you created and configured a SQL Database and firewall rules.
