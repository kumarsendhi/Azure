Exercise 5: Implement Redis Cache

Overview

In this exercise, you will configure Redis Cache for your web application deployed in Azure.

Time Estimate

30 Minutes

Task 1: Create the Redis Cache Instance

In this exercise, you will create a Redis cache instance and instrument your application to use it.

1.     Log into the Azure portal (https://portal.azure.com).

2.     Click Resource groups and open the EventsDB_RG resource group. Then click the ADD button.

3.     Type Select Redis Cache in the Search box and choose Redis Cache and click Create.


4.     Specify a unique name for the cache. Change the location to match the location of your website and database and then click Create.

 

Note: It may take several minutes for provisioning to complete.

Task 2: Install Assemblies and Configure the Connection

1.     Within the Events project in Visual Studio open the Package Manager Console and execute the following command.

 

2.     Type the following command Install-Package StackExchange.Redis and hit Enter.

Install-Package StackExchange.Redis


 

3.     If prompted, choose Yes to reload Web.config

 

4.     Once the deployment of the cache is complete, open the Redis cache Settings in the Azure portal by clicking Resource groups, opening the EventsDB_RG resource group and then clicking the Redis cache that was just created

 

5.     In the Settings blade, click Access keys to retrieve the authentication keys

 

6.     Construct a connection string using the cache name and the primary key in the following format. Replace the placeholder values with the values for your cache.

 

[cache name].redis.cache.windows.net,ssl=true,password=[primary key]

Example:  eventscache101.redis.cache.windows.net,ssl=true,password=bJ0TlAa/kS/XsyGvpk+vXGE394UO262uBH4bZYPEXH4=

 

7.     Create a new connection string in the web.config of the Events application.

name = RedisConnection and connectionString = [the connection string].  You do not need a providerName attribute.

8.     Save the change to web.config.

 

9.     Add the Redis .NET extensions within Visual Studio, click the refresh button to show the new folder.  Right click on the RedisExtensions folder and click Include in Project.

 

 

 

10.  Using Visual Studio add the following namespaces to the global.asax.cs files.

using System.Configuration;

using StackExchange.Redis;

 

 

11.  Add the following static property to the global.asax.cs file.

private static ConnectionMultiplexer redisCache;

public static ConnectionMultiplexer RedisCache
{
    get
    {
        if (redisCache == null || !redisCache.IsConnected)
        {
           redisCache = ConnectionMultiplexer.Connect(ConfigurationManager.ConnectionStrings["RedisConnection"].ConnectionString);
        }
        return redisCache;
    }
}

12.  Save the change to Global.asax.cs.

Task 3: Instrument the Application using the Redis Cache

1.     Open the \Repository\EventsAppService.cs file.

2.     Add the namespaces for Redis to the top of the page under the existing namespaces.

using StackExchange.Redis;
using Events.RedisExtensions;

 

3.     Update the code to use the Redis cache.
Find the function:

    public IEnumerable<Event> GetUserEvents(string activeDirectoryId)

    {

            return this._repository.GetUserEvents(activeDirectoryId);

    }

Replace it with the following:

    public IEnumerable<Event> GetUserEvents(string activeDirectoryId)

    {

        IDatabase cache = MvcApplication.RedisCache.GetDatabase();

        var events = cache.Get<IEnumerable<Event>>(activeDirectoryId);

        if (events == null)

        {

           events = this._repository.GetUserEvents(activeDirectoryId);

           cache.Set(activeDirectoryId, events, TimeSpan.FromMinutes(5));

        }

            return events;

    }

Note: This code attempts to retrieve the data out of the cache. If it is not found, it is retrieved from the database, and then stored in the Redis cache for 5 minutes.

 

4.     Rebuild the application and test locally by viewing in the browser.  The application will now cache the results with Redis for 5 minutes before retrieving from the database again.

 

5.     Publish the project to Azure, by right-clicking on the project and choosing Publish.

6.     Click the Publish button again to update the web app.

Summary

In this exercise, you will have configured and tested Redis Cache for your web application deployed in Azure.
