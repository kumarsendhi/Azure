Exercise 6: Azure Web App Diagnostics

Overview

In this exercise, you will learn how to configure Azure Web App Diagnostics and remote debugging with Visual Studio.

Time Estimate

30 Minutes

Task 1: Enable Website Logging

1.     From Visual Studio 2017, click View -> Server Explorer and connect to your Azure subscription if it is not connected.

 

2.     Right click on the web app and click View Settings

 

3.     Change Web Server Logging to ON and click Save.

 

4.     Right click on web app and click View Streaming Logs

 

5.     Click the gear icon to configure streaming logs.

 

6.     Uncheck application logs, check web server logs and click OK.

 

7.     Press the Start Monitoring real time Azure logs button to start monitoring.

 

8.     Browse through the application and you should see web server logs in streaming logs window.

 

9.     Right click on web app and click Stop viewing Logs

 

Task 2: Enable Application Logging

1.     From Visual Studio 2017, Open Server Explorer and connect to your Azure subscription.

 

2.     Right click on web application and click View Settings

 

3.     Change Application Logging to Verbose and click Save.

 

4.     Open HomeController.cs file and add following lines inside the Index() method (right before the return statement).

  System.Diagnostics.Trace.TraceInformation("Informational trace");

  System.Diagnostics.Trace.TraceError("Error trace");

  System.Diagnostics.Trace.TraceWarning("Warning trace");

 

5.     Rebuild and publish the web application.

 

6.     Right click on web app and click View Streaming Logs

 

7.     Click the gear icon to configure streaming logs.

 

8.     Check application logs, uncheck web server logs and click OK.

 

9.     Browse through the application and you should see application logs in streaming logs window with all three of the trace information (and a bunch more).

 

10.  Change the application logging setting to Error and refresh the page. You should only see error trace messages now.

 

11.  Right click on web app and click Stop viewing Logs

 

Task 3: Enable Remote Debugging

1.     From Visual Studio 2017, Open Server Explorer and connect to Azure subscription.

 

2.     Right click on web application and click View Settings

 

3.     Change Remote Debugging to ON and click Save.

 

4.     Right click project and choose to publish. Click Settings.

 

5.     On the Publish dialog, click Settings and then change the configuration to Debug so the release will have debugging symbols included.

 

 

6.     Click Save and then Click Publish.

 

7.     Open Controllers\HomeController.cs and add a break point to the first line (place the cursor and press F9).

 

8.     In Server Explorer, right click on the web app and click Attach Debugger.

 

9.     Browse to home page of the application. You will be able to step through the code now from attached debugger.

Summary

In this exercise, you have learned how to enable logging for your web app and perform remote debugging of azure web app.
