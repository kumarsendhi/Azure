Exercise 4: Create the First Virtual Machine

In this exercise, you will deploy a virtual machine in a virtual network, that uses network security groups to only allow web traffic for security purposes and that is deployed into an availability set for additional durability.

Task 1: Deploy WebVM-1

1.     Click New, Virtual Machines, and then select the Windows Server 2012 R2 Datacenter image from the FEATURED APPS list, and then click Create.

 

 

2.     Specify the following configuration and click    .

·       Name: WebVM-1

·       VM disk type: SSD

·       User name: demouser

·       Password: demo@pass123

·       Subscription: Ensure the correct subscription is selected

·       Resource Group: Create new - OpsVMRmRG

·       Location: the same region selected for the Azure Virtual Network

 

3.     Choose DS1_V2 Standard and then click Select at the bottom of the page.

 

4.     Specify the following configuration options:

·       Virtual network: Click the Virtual Network tile to change it to OpsTrainingVNET (if it is not already selected).

·       Subnet: Click the Subnet tile to change it to the Apps subnet (if it is not already selected)

·       Click the Network security group tile (see next step)



5.     Change the name of the network security group to APPSNSG.

 

6.     Click the Add an Inbound Rule link and specify the following configuration:

·       Name: HTTP

·       Priority: 100

·       Source: select Any

·       Service: HTTP

·       Action: select Allow

 

 

7.     Click OK to accept these choices.

 

8.     Scroll down and click the Availability set tile.

 

9.     Click Create new, and name the availability set WebAVSet and then click OK.

 

10.  Click OK until all blades are closed and the virtual machine starts to provision.

 

11.  Wait until the first virtual machine is provisioned before continuing.
