Exercise 5: Create the Second Virtual Machine

In this exercise, you will deploy a second virtual machine in the same virtual network, using the same network security group, and the same availability set.

Task 1: Deploy WebVM-2

1.     Click New, Virtual Machines, and then select the Windows Server 2012 R2 Datacenter image from the FEATURED APPS list and then click Create.

 

2.     Specify the following configuration and click    .

·       Name: WebVM-2

·       VM disk type: SSD

·       User name: demouser

·       Password: demo@pass123

·       Subscription: Ensure the correct subscription is selected

·       Resource Group: Use existing - OpsVMRmRG

·       Location: the same region selected for the Azure Virtual Network

 

3.     Choose DS1_V2 Standard and then click Select at the bottom of the page.

 

4.     Specify the following configuration options:

·       Select the existing OpsTrainingVNET virtual network

·       Select the existing Apps subnet 

·       Select the existing APPSNSG network security group

·       Select the existing Availability set WebAVSet



 

5.     Click OK until all blades are closed and the virtual machine starts to provision.

 

6.     Wait until the virtual machine is provisioned before continuing.
