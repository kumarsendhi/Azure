Exercise 6: Configure and Test a Web Server

Task 1: Install IIS on both virtual machines

In this exercise, you will install the Internet Information Server (IIS) web server on the two virtual machines and modify the default HTML returned so you can validate load balancing in a later exercise.

1.     Within the Azure Management Portal, click Virtual Machines.

 

2.     Click WebVM-1 to open the configuration.

 

3.     Connect to the WebVM-1 virtual machine by opening the virtual machines configuration and clicking the connect button and when prompted logging in with the credentials:

·       User Name: demouser

·       Password: demo@pass123

 

4.     Once logged in click the PowerShell Icon on the task bar then execute the following PowerShell command.

Install-WindowsFeature "Web-Server"

 

5.     Once IIS is installed on the servers navigate to the file iisstart.htm under the folder c:\inetpub\wwwroot.  This folder is automatically created when the web-server role is installed.

 

 

 

6.     Now you must edit the default iisstart.htm file by right clicking then select open with notepad.

 

7.     Once notepad is open, make a new line under the <div id="container"> line. Then add the server name (WebVM-1) followed by the <br /> tag, as shown in the below screenshot and save it. This change will allow you to see which server the request is currently being served from.

 

8.     Repeat these on the WebVM-2 virtual machine as soon as it has provisioned. Make sure you specify WebVM-2 as the server name.
