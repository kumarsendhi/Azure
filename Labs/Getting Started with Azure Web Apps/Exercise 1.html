Lab Overview

In this lab, you will create an Azure Web App and a SQL Database and configure the popular content management system (CMS) Orchard CMS.  You will then configure the web app to automatically scale based on actual CPU usage.

Time Estimate

·       60 minutes

 

Exercise 1: Deploy a new Blog Website using Orchard CMS and Azure

In this exercise, you will deploy the popular Content Management Suite Orchard in Azure by creating an Azure Web App and a SQL Database to store its data.

 

Task 1: Deploy the Web App and SQL Database

1.     Launch a browser and navigate to http://portal.azure.com and login with your Azure subscription credentials.

 

2.     Within the Azure Management Portal, click New.

 

3.     Type in web + sql and press enter.

 

4.     Click on Web App + SQL and then click Create on the next screen.

 

 

5.     Specify a unique name for the App name field. Ensure the green check mark appears which tells you the name is valid. Specify OrchardWebRG for the resource group name.

Note: The resource group will be used to group your web app and SQL Database in the same logical unit for billing and management.

 

6.     Click the App Service plan/location tile.

 

7.     Click Create New to create a new App Server plan.

 

8.     Specify the following configuration:

·       App Service plan: MyPlan

·       Location: Choose the closest region to you

·       Pricing tier: Click to view the available options that you can scale your web app too.

 

 

 

9.     On the Pricing tier page, you choose the hardware and features your web app will run on. If you click View All you can see the additional options. Choose the S1 Standard plan and click Select and then click OK to go back to the creation blade.

 

10.  On the Web App creation blade click the SQL Database tile.

 

11.  Click the Create a new database tile.

 

12.  Name the database OrchardDB and click Target server (Ensure that SO Standard is selected for the pricing tier).

 

13.  Specify the following configuration for the SQL Database Server. 

 

·       Server name: enter a unique name

 

·       Server admin login: demouser

 

·       Password: demo@pass123

 

·       Confirm Password: demo@pass123

 

·       Location: choose the same region used for the Azure Web App

 

14.  Change the Applications Insights switch to On. Application Insights is a service that allows you track usage and issues with your application code.

 

15.  Check Pin to dashboard and click the Create button to create the Azure Web App and SQL Database with the Orchard Content Management System enabled.

 

Task 2: Deploy the Orchard Software from GitHub

1.     After the web app has finished provisioning, click the Deployment options link.

 

2.     Click Choose Source, and click on External Repository.

 

3.     Specify the public orchard repository in the Opsgility GitHub organization for the Repository URL field

https://github.com/opsgility/orchard and then click OK.

 

 

Note: It will take several minutes for the Orchard application to be built and deployed. You can click the Deployment Options link again to view the progress. Once the deployment is marked Active you can continue.

 

Task 3: Connect to the Database

1.     Open the web app, by clicking Resource Groups on the left and open the OrchardWebRG resource group and then clicking your web app.

 

2.     Click the Application Settings tile on the left navigation for the web app.

 

3.     Click the Show connection string values link to show the connection string to the SQL Database.

 

4.     Click inside the textbox, and press CTRL-A to select the entire value. Then press CTRL-C to copy it to the clipboard.

 

5.     Click on the Overview tile on the web app, and then click the URL link on the essentials pane to open the web app.

 

6.     Specify the name of the Orchard Site, and use the same password demo@pass123 used previously (type in the password  do not use the clipboard).

 

7.     Specify the option to Use an existing SQL Server, SQL Express Database and paste in the connection string copied to the clipboard earlier.

 

 

8.     Change the Orchard Recipe to Blog, and click Finish Setup.

 

9.     You have now successfully deployed your own blog in Microsoft Azure!
