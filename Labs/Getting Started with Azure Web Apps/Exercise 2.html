Exercise 2: Configure Automatic Scaling

In this exercise, you will learn how to automatically scale your Azure Web App in the event your new blog site grows in popularity due to your keen writing skills.

Estimated Time

15 minutes

 

Task 1: Configure Auto Scale

1.     In the Azure Management Portal in your Web App Settings, scroll down on the left navigation and click the Scale Out (App Service Plan) tile.

 

2.     Click the Enable autoscale button,

 

3.     Set the Minimum instance count to 2, the Maximum to 5, and the Default to 2 then click the Add Rule button to configure the criteria for adding and removing instances.

 

4.     Review and accept the default criteria by clicking the Add button.

The default criteria will add instances if your CPU utilization goes above 70% for longer than 10% minutes Azure will until the spike is mitigated. Once it falls below it will automatically remove instances (every 5 minutes) until the minimum of 2.

 

5.     Name the Autoscale setting name: OpsAutoScale.

 

6.     Click the Save button to save the autoscale configuration.
