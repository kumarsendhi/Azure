Overview

In this Azure Automation Lab, you will configure and deploy a fully automated environment with source control and module integration. Through this environment, you will gain the ability to centrally manage you’re on-premises infrastructure, the public cloud in Azure as well 3rd party providers such as Amazon Web Services. You will also configure and deploy the Operations Management Suite for monitoring and change tracking. This will enable you to gain telemetry from a centralized console to take actions through automation based on repeated events and security alerts empowering you to be proactive instead of reactive. You will also gain the ability to have source control over yourself and manage users locally with their machines through Azure PowerShell and within the Azure Platform. As you progress throughout the Exercise your LABVM is where the execution of scripts written and confirmation will take place. WEBVM-1 and WEBVM-2 will be used to validate your autonomous configuration through DSC and Integration Modules. The Hybrid Worker VM will serve as the connecting point for you to leverage runbooks within the Automation account against resources in your local datacenter and 3rd party Cloud providers.

Objectives

  • Describe Azure Automation components, including runbooks, Webhooks, schedules, assets and modules.
  • Implement and manage an autonomous solution within Azure and connected on-premises environments.
  • Plan and create Azure an Automation Solution.
  • Configure, manage, and monitor Azure resources to optimize availability and reliability.
  • Deploy and configure web servers using Desired State Configuration (DSC), solution to multiple servers at a time centrally managing their compliance and status.
  • Implement, manage, and monitor resources with automation.
  • Plan and implement data services based on SQL Database to support applications.
  • Integrate on-premises infrastructure with 3rd party cloud providers
  • Automate operations in Azure management by using automation.
  • Enable a DevOps solution to ensure integrity of code and proper deployment
  • Module creations to deploy resources to teams securely and with the user of least privileged in mind
  • Automate task based on schedules, process request, alerts or security thresholds being triggered.
  • Leverage the PowerShell community gallery to import and export solutions you see beneficial to and to publish solutions you have created yourself.

What you will build

Exercise 1: Setup a Lab Machine

Overview

In this exercise, you will create a Virtual Machine (VM), in Azure that will be used to complete the exercises in this course. After the LABVM has been provisioned you will download the Student Files that will be used for this Lab, and install the Azure PowerShell Cmdlets.

Note: Skip this exercise if you are using Opsgility Real Time Labs

 

Task 1: Login to the Azure Management Portal

1.     Open your browser and navigate to http://portal.azure.com

 

2.     Enter the account associated with your Microsoft Azure subscription.  If prompted, choose whether your account is an organization account or just a Microsoft Account.

 

 

3.     Click on +NEW, and in the search box type in Windows Server 2016 Datacenter and press enter.

 

4.     In the returned search results click the Windows Server 2016 Datacenter image.

 

 

5.     In the blade that comes up, at the bottom of the page the deployment mode should be set to Resource Manager, then click Create.

 

6.     Set the following configuration on the Basics tab and click OK.

a.     Name: LABVM

b.     VM disk type:  SSD

c.      User name: demouser

d.     Password: demo@pass123

e.     Subscription:  Select the Subscription you wish to use

f.       Resource Group: OPSEXERCISERG

g.     Location: EAST US

 

 

7.     On the Size blade, choose the DS1_V2 Standard instance size.

Note:  You may have to click the View All link to see the instance sizes.

 

8.     On the Settings blade choose Yes for use Manged Disks.

 

9.     Accept the default values for the rest of Settings blade and click OK.

 

10.  Review the selections on the Summary blade then click OK. The deployment should begin provisioning. It may take 10+ minutes for the virtual machine to complete provisioning.

 

 

Note:  Wait for the LABVM to complete prior to moving on to the next Exercise.

 

Task 2:  Download and Unzip Student Files

 

1.     From the Azure Portal page locate the OPSEXERCISERG where the LABVM should now show it is Running.  Click LABVM and then click Connect to connect via Remote Desktop.

 

2.     An RDP file will be downloaded to your local computer.  Locate this file in your downloads folder and open it to connect to LABVM.

 

3.     The Windows Security box will appear, click the Use a different account and then enter the following credentials:

a.     User name: demouser

b.     Password:  demo@pass123

 

4.     You will be presented with a Remote Desktop Connection warning because of a certificate trust issue. Click Yes to continue with the connection.

 

5.     When prompted about Networks, click No.

 

6.     The Server Manager will load.  Click Configure Local Server and then on the right side of the pane, click On by IE Enhanced Security Configuration.

 

 

7.     Change the Administrators to Off and then click OK.

 

8.     Open Internet Explorer, Click Ok to accept the Recommended Security settings and then download the Student Files by using Lab Support Files found on this lab page on Opsgility.com

 

9.     Click Save when prompted.

 

10.  Once the file is downloaded IE will show the operation as completed.  Click Open Folder.

 

11.  Right-click the Opsgility Automation Resources.zip file and then click Extract All.

 

12.  Change the folder to C:\OpsgilityTraining and click Extract.

 

 

Task 3:  Install the Azure PowerShell Cmdlets

 

1.     From within the LABVM point the here browser to this will download the Web Platform installer.  Click Run.

 

2.     The Azure PowerShell installer dialog box will appear.  Click Install.

 

3.     On the next dialog, click I Accept to continue with the installation.

 

4.     When the install is completed, click Finish and then Exit.

 

5.     Close, the RDP Session and then from the Azure Portal click Restart.

 

Summary

In this exercise, you created a VM in Azure that will be used to complete the exercises in this course. After the LABVM was provisioned you downloaded the Student Files and installed the Azure PowerShell Cmdlets.
