Exercise 2:  Create an Azure Automation Account

Overview

Through Azure Automation you gain the ability to execute Runbook automations, DSC configurations/deployments and the ability to create webhook, schedules, assets and integration with Source control. In this exercise, you will configure an Azure Automation account as well as an OMS Workspace.  Then you import a script from the PowerShell Runbook and run the script.

Time Estimate

30 Minutes

Task 1:  Create an Azure Automation Account

In this task, you will create an Azure Automation Account that will be used as the basis for the remaining exercises in this course.

1.     From the LABVM connect to Azure by visiting https://portal.azure.com. Login and from the Azure Dashboard locate the OPSEXERCISERG resource group.  From within OPSEXERCISERG click +Add.

 

2.     Type Automation & Control in the search box and press enter. Select Automation & Control then click the Create button to start the configuration.

 

3.     On the Automation and Control Blade Select OMS Workspace.

 

4.     Then select Create New Workspace. Enter a unique name for the OMS Workspace Name. Select your subscription for the deployment. Under Resource group choose Use existing and select OPSEXERCISERG from the list. Leave the Location as East US and the Pricing tier at Free.

 

5.     Select the Automation account

 

6.     Select Create Automation Account.

 

7.     Enter the Name Lab-Deployment. The remainder of the options will be grayed out. Click the OK button twice allowing each service to validate before moving on. When back on the Automation & Control blade check the box Pin to dashboard and then click the create button to launch the deployment.

 

8.     Your Automation Account and OMS Workspace is now deploying and will only take a few moments.

 

9.     Once the OMS Workspace has been deployed, clicking Create to complete the Final deployment for Automation & Control.

 

NOTE:  Please wait until the deployment is complete prior to moving on to the next Task.

 

Task 2:  Importing Scripts from the Gallery

In this task, you will view the gallery both Microsoft and Community to import a simple Output script to run and publish.

1.     From within the OPSEXERCISERG locate the new Lab-Deployment Automation Account and click to open.

 

2.     From the Lab-Deployment account select runbooks.

 

3.     Click on the Browse gallery link at the top of the screen.

 

4.     The gallery is presented with different types of scripts. PowerShell scripts, Graphical Runbooks and PowerShell workflow are all present both from Microsoft and the Community.

Note: Take the time to view some of the options that are in the Gallery to familiarize yourself with the many options.

 

5.     In the search box type hello world and press enter. Both a PowerShell Script and Graphical Runbook will be presented. Select the PowerShell Workflow Runbook.

 

6.     The blade will fly out allowing you to view the source project. Select the import button at the type.

 

7.     The import blade will appear.  Enter Write-HelloWorldDemo for the Name. Notice how the Runbook Type is grayed out. Because it is an import you cannot configure it further. Select OK to import the Runbook.

 

8.     After a successful import of the script you can see the Runbook details. From the Write-HelloWorldDemo runbook select Edit.

9.     The Edit blade extends out presenting the workflow of the Runbook, now click Test Pane.

 

10.  The Test blade extends presenting the parameter Name that must be filled. Enter World and then click the Start button for your runbook to be processed and to return he output Hello World.

 

Note: Azure Automation can be executed both in Azure and on-premises. Since you are getting started we only have the capability to run in Azure. Later in this lab you will configure Hybrid Runbooks that would run outside of Azure.

Summary

In this exercise, you configured an Azure Automation account as well as an OMS Workspace.  You then imported a PowerShell Runbook from the gallery and ran it against your environment.
