Exercise 3:  Creating Automation Assets for Deployments

Overview

In this exercise, you will configure a variable asset to execute against a VM that you will create using Azure PowerShell ISE. You will then create a graphical runbook configuring the parameters to execute the shutdown of the newly created VM. The purpose of this Exercise is the detail the steps for higher automation of larger environments where the machines need to be systematically shutdown.

 

Time Estimate

60 Minutes

Task 1:  Deploy VM in Azure to use for this Exercise

In this task, you will create a VM in Azure to use for the exercise.

1.     From the LABVM open the Azure Portal in Internet Explorer.

 

2.     Click Resource Groups > +Add and Name the Resource Group: OGDEMOTM, choose your Subscription and then Select East US for the Region.  Then click Create.

 

3.     From within the newly created OGDEMOTM RG, click +Add

 

4.     In the Filter box type: Template Deployment and press enter.  Then click Template Deployment.

 

5.     Press Create and then on the Customer Deployment Blade click Edit.

 

6.     Click the Load File button.

 

7.     Locate the WebVMUS.json file which is in the C:\OpsgilityTraining\Opsgility Automation Resources folder on your LABVM.

 

8.     This will load the ARM template, click Save.  Complete the Settings with these values, click the I Agree and Purchase buttons.

a.     Subscription:  Choose your subscription.

b.     Resource Group: OGDEMOTM

c.      Admin Username: demouser

d.     Admin Password:  demo@pass123

 

 

9.     The deployment will be submitted to Azure to build the WEBVMUS along with their associated resources in the OGDEMOTM Resource Group.

 

NOTE:  Wait for the Deployment to complete prior to moving on to the next steps.

 

Task 2:  Create a Variable Asset

In this task, you will create an Azure Automation Asset that will automate the shutdown of a VM.

1.     In the azure portal, https://portal.azure.com. Select your Lab-Deployment Account and then under the settings blade select Assets.

 

2.     From the Assets menu, click Variables.

 

3.     Click +Add a variable

 

4.     The purpose of this is to provide the Automation account to the Runbook so it can authenticate to execute against your infrastructure. Update the Blade by Entering the following information and then click Create.

a.     Name:  AzureSubscriptionID

b.     Type:  String

c.      Value:  Enter you Subscription ID number

d.     Encrypted: No

NOTE: You can retrieve your Subscription ID by running the following commands in a PowerShell window on your LABVM:

LoginAzureRmAccount

Get-AzureRmSubscription

 


Task 3:  Create a Graphical Runbook

In this task, you will create a Graphical Runbook to shut down the virtual machine you just created.

1.     Back to your Azure Automation Account Lab-Deployment select Runbooks.

 

2.     Click +Add a runbook.

 

3.     Next, Quick Create, Create a new runbook.

 

4.     Enter Shutdown-VM as the name and select Graphical Runbook for the type and then click Create.

NOTE:  Make sure there are no leading or trailing spaces in the name of the Runbook or it wont save.

 

5.     In the library search for Add-AzureRm. You will find it under CMDLETS. Right click Add-AzureRmAccount and then select Add to Canvas.

 

6.     Next in the library search for AzureRunAs and then locate AzureRunAsConnecton using Right-click again click Add to canvas.

 

NOTE:  Now, that these two Cmdlets have been added to the canvas they now need to be configured. 

 

7.     Hover over Get-AutomationConnection until a circle appears on the bottom of the shape. Click the circle and drag the arrow to Add-AzureRmAccount. The arrow that you just created is a link. The runbook will start with Get-AutomationConnection and then run Add-AzureRmAccount.

 

8.     On the canvas, select Add-AzureRmAccount and in the Configuration control pane type Update the Label to read Sign In to Azure.

 

9.     Click Parameters and the Activity Parameter Configuration blade will appear.

 

10.  Click Parameters Set, choose a parameter set.

 

11.  From the list click ServicePrincipalCertificate.

 

12.  Once you select the ServicePrincipalCertificate parameter set, the parameters are displayed in the Activity Parameter Configuration blade.

 

13.  Click the APPLICATIONID parameter.

 

14.  In the Parameter Value blade, select Activity output for the Data source and select Get-AutomationConnection from the list, in the Field path textbox type ApplicationId, and then click OK. We are specifying the name of the property for the Field path because the activity outputs an object with multiple properties.

 

15.  Next click the CERTIFICATETHUMBPRINT parameter.

 

16.  Select Activity Output for the Data source, select Get-AutomationConnection from the list, in the Field path textbox type CertificateThumbprint, and then click OK.

 

17.  Click SERVICEPRINCIPAL

 

18.  Select ConstantValue for the Data source, click the option True, and then click OK.

 

19.  Now, click TENETID Parameter.

 

20.  Select Activity output for the Data source. Select Get-AutomationConnection from the list, in the Field path textbox type TenantId, and then click OK twice.

 

 

21.  In the Library control, type Set-AzureRmContext in the search textbox and then Add to canvas.

 

22.  Select Set-AzureRmContext and in the Configuration control pane type Specify Subscription Id in the Label textbox.

 

23.  Click Parameters and the Activity Parameter Configuration blade appears.

 

24.  Click parameter set, choose a parameter set.

 

25.  Click SubscriptionID.

 

26.  Click SUBSCRIPTIONID.

 

27.  Under Data Source, select Variable Asset.  Then the AzureSubscriptionID variable you created earlier will appear.  Click AzureSubscriptionID and then OK twice.

 

28.  Back on the Canvas, hover over Sign in to Azure until a circle appears on the bottom of the shape. Click the circle and drag the arrow to Specify Subscription Id.

 

29.  Next you will add the Stop-AzureRmVm cmdlet to stop a VM. 

 

30.  In the Library control, type Stop-AzureRmVm and then Add to canvas.

 

31.  Click and drag it under the Specify Subscription Id.

 

32.  Hover over Specify Subscription Id until a circle appears on the bottom of the shape. Click the circle and drag the arrow to Stop-AzureRmVM.

 

 

33.  Select Stop-AzureRmVM on the canvas. Click Configure Parameters and then Parameter Set to view the sets for Stop-AzureRmVM.

 

 

34.  Select the ResourceGroupNameParameterSetName parameter set.

 

35.   Select Name.

 

36.  For the Data source select PowerShell expression and type WEBVMUS surrounded with double quotes that will stopped by this runbook. Click OK.

 

37.  Select the RESOURCEGROUPNAME parameter.

 

38.  Use PowerShell expression for the Data source and type in the name of the resource group OGDEMOTM, surrounded with double quotes. Click OK twice.

 

39.  Next, click the Optional additional parameters Configure parameters.

 

40.  Type -Force, and Click OK.

Note:  This -Force command must be entered for this RunBook to Execute.

 

 

41.  The Canvas should now resemble this screen capture.

 

42.  In another Browser Tab login to the Azure Portal and make sure that the VM you specified in this Run Book is currently Running.  If not, start the VM and make sure it is up and running prior to moving to the next step.

 

43.  Click Test pane so that we can test the runbook.

 

44.  Click Start to begin the Test.

 

45.  After the runbook completes your output should be successfully completed as shown below.

 

46.  Now, that the Runbook has been verified to run properly it should be published.  To do, simply press Publish and click Yes.

 

Task 4:  Adding Input Parameters to the Shutdown-VM Runbook

In this task, the Runbook will be updated to allow for the user to input the name of the VM to be shut-down rather than the hard coded VM name as it currently stands. To solve this, you will add an input parameter allowing the user of the runbook to enter the values of the VM without changing the runbook configuration itself.

1.     Select Runbooks from your Automation Account Lab-Deployment.

 

2.     Click the Shutdown-VM runbook and then Edit on the menu screen to bring the canvas back up.

 

3.     Next select Input and Output at the top of the menu.

 

4.     Click +Add input

 

5.     In the Name field enter VMNAME. For the Type enter String and select Yes for Mandatory. Then click OK.

 

6.     Select +Add Input again this time Adding ResourceGroupName for the Name field, Type as String and Mandatory as Yes. Select Ok and your screen should be the same as below.

 

7.     Select OK once more and then click Save in the upper left hand corner of the menu.

 

8.     Next select Stop-AzureRmVm from the canvas.

 

9.     Select Parameters and then select Name under Parameter Sets.  Change the Data Source from PowerShell expression to Runbook input and then Select VMNAME and Click OK.

 

 

10.  Select ResourceGroupName and then change the Data source from PowerShell expression to the Runbook input. This time select ResourceGroupName then select OK twice bringing you back to the canvas.

 

11.  Again, click Save on the Runbook.  Then click Publish and Yes to overwrite the previous version.

 

12.  Using the Azure Portal make sure that the WebVMUS VM is running to test the newly updated Runbook.  Once the VM is verified to be Running move on to the next step.

 

13.  In the Azure Portal locate the Shutdown-VM in the OPSEXERCISERG.

 

14.  Open the Shutdown-VM Runbook and then click Start.

 

15.  This time when the Runbook is started now, the user will be prompted to enter in the name of the VM and the Resource Group in which it resides.  Enter WebVMUS and OPSDEMOTM

 

16.  After entering in the Parameters, the job will be submitted by clicking OK.

 

17.  The Job Window will appear in the Portal.  Click the Output window to see the results of the Runbook.

 

 

18.  Once the job is completed the VM will be deallocated.  This can be confirmed using the Azure Portal.

 

Summary

In this exercise, you configured a variable asset to execute against a VM that you created using Azure Automation Service Account. You then created a graphical runbook configuring the parameters to execute the shutdown a VM.
