Exercise 4:  Setting Up Source Control

Overview

In this exercise, you will configure Source Control for your Azure Automation Account. The purpose of this Exercise to help familiarize you with continuous integration, DevOps direction and history tracking and centralization of your development assets.

Time Estimate

30 Minutes

Task 1:  Add a GitHub Account

In this task, you will create a GitHub account for source control, management and deployment.

1.     Browse to https://github.com/ and either create a new or login to your existing GitHub account.

 

2.     On the GitHub homepage, click "Create a repository." You can also choose an existing repository but for this walkthrough, we’ll create a new one.

 

3.     Give the repository the name AzureAutomation, select Public, check Initialize this repository with a README and then click "Create repository".

 

4.     Now that your repository is created in the Azure Portal locate the Azure Automation Account Lab-Deployment. Under Account Settings click Source Control.

 

5.     Next, choose source Configure required settings.

 

6.     Select GitHub.

 

7.     Click Authorization, Configure required settings.

 

8.     Click Authorize, and then when the GitHub window appears click Authorize Application.

 

9.     Once the Portal shows that the authorization process is completed, click OK.

 

10.  Click Choose repository, configure require settings and then Select the AzureAutomation repo that is located on GitHub.

 

11.  Next, choose branch and select master.

 

12.  Select Runbook folder path, and put in “/” as the path (remove the quotes).

 

13.  Review and then click OK.

 

14.  Now, back on the Lab-Deployment Azure Automation account click Source Control again.  The GitHub Account will now appear.

 

15.  Click Sync to have the Runbooks in your account put into the GitHub repo. 

 

16.  You will need to Say Yes to sync all the runbooks.

 

17.  Once completed you can now check in your runbooks as well as pull them from GitHub.

 

18.  From the Azure Automation account click Runbooks, and then +Add a runbook.

 

19.  Select quick create.

 

20.  In the Name field enter SourceTest and select PowerShell for the Runbook Type. The select the create button.

 

21.  Your Runbook is now created and notice there is now a Check In button option on the menu.

 

22.  In the Script pane type the following PowerShell Code.

Write-Output “Azure Automation Rocks!”

 

23.  Next click the Check In button.

 

24.  Click Yes, on the Check-in message This will check the updates to this runbook into your source control repository.

 

25.  Next Click Publish and Yes to Publish Runbook.

 

26.  Select the Start and then Yes to confirm running the Runbook.

 

27.  Using your browser confirm that the file has been uploaded to the AzureAutomation repo.

 

28.  Click the SourceTest.ps1 file to review it in GitHub.

 

29.  When the details page for SourceTest.ps1 opens click Edit.

 

30.  Update the code in this file with the following:

Write-Output "Azure Automation Rocks Harder - with GitHub!"

 

31.  Scroll down on the page after making the changes to the code verify that the radio button next to Commit directly to the master branch is selected.  Click the Commit Changes button. 

 

32.  Move back to the Azure Automation account and click Source Control > Sync.

 

33.  Click on this job that shows as Starting or Running and then click the Output button.

 

34.  The job will show the Runbooks that were updated.

 

35.  Close the Output blade, and click Runbooks > SourceTest.

 

36.  Click the </>View button on the SourceTest Runbook. 

NOTE:  Notice that the code shown doesn’t include the updates yet.  This is the “Published Runbook”.  This ensures that until you are confident in your code it won’t be promoted to “Production”, until you take that step.

 

37.  Close the View blade and click Edit on the Runbook.

 

38.  On the Edit Menu, there are a few options here.  Publish will commit it to production, Revert to Published would rollback the changes, Check In would commit it back to GitHub and the Test Pane.

 

39.  You can click the Test Pane > Start to perform a quick Test.  Once the Test completes successfully then close the Test pane.

 

40.  Click Publish and confirm with a Yes to commit to production.

Summary

In this exercise, you configured Source Control for your Azure Automation Account. You then created a simple PowerShell Runbook which was checked in for source control, version history, management, and security. This is the start to enabling you to configure and deploy a DevOps environment.
