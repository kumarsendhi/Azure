Exercise 5:  Creating a PowerShell Runbook using PowerShell ISE

Overview

In this exercise, you will configure the Azure Automation Authoring Toolkit to add on to PowerShell ISE within the LABVM if created or on your local machine if you decided to install Azure PowerShell on your local box. You will also create a credential asset to use for authentication to the account for script execution and source control. 

Time Estimate

30 Minutes

Task 1:  Create a Credential Asset

1.     From the Azure Dashboard select More Services at the bottom of the Left hand navigation pane, and in the Filter box enter Azure Active Directory and select it to open it up.

 

2.     Click Users and Groups.

 

 

3.     Click +Add.

 

4.     Provide the Name AutoDeploy which you will use for authentication. Next provide a User Name which should be autodeploy@<directoryname>.onmicrosoft.com. Click the Show Password and take note by copying to the notepad, then click Create.

Note:  To find the name of the directory it should be directly to the right of the Microsoft Azure logo on this screen.

 

5.     Open an In-Private session in Internet Explorer and point the browser to http://portal.azure.com.

6.     When prompted use these credentials:

a.     User Name:  autodeploy@<yourdirectory>.onmicrosoft.com

b.     Password:  Paste the Password from the Notepad

7.     Azure will force an update the password.  Use this information to update:

a.     Current Password:  Paste the Password from the Notepad

b.     New Password:  demo@pass123

8.     Close out of the Azure AD Blade.  In Resource Groups click the OPSEXERCISERG and click the Lab-Deployment Azure Automation Account.

 

9.     On the Overview page of Lab-Deployment click Assets.

10.  Next click Credentials.

 

11.  Select the +Add a credential.

 

12.  For the Name use AutoDeploy and enter autodeploy@<yourdomain>.onmicrosoft.com, and then the updated password demo@pass123. Click Create.

 

 

13.  Next give AutoDeploy permission to the Azure Automation Account. From the Azure Portal click Resource Groups > OPSEXERCISERG.

 

 

14.  Select Access Control (IAM) then Add at the top.

 

15.  Click +Add.

 

16. In the Select a role blade click Owner.

 

17. From the Add users click the AutoDeploy account. This will allow this user to gain access not only to the Lab-Deploy account and everything else in the OPSEXERCISERG resource group.  Click Select > OK.

Task 2: Configuring PowerShell ISE for Azure Automation Authoring

 

1.     From a RDP session to LABVM open PowerShell ISE by selecting the search glass and typing PowerShell.

 

2.     In the PowerShell, interactive window run the following command.  When prompted click Yes to All.

Set-ExecutionPolicy -ExecutionPolicy Unrestricted -Scope CurrentUser

 

3.     From PowerShell ISE click File > Open locate C:\OpsgilityTraining\Opsgility Automation Resources then click the file named Install Azure Automation Authoring Toolkit

 

4.     The script will open, then click the Play button to execute the script. Click Run Once at prompt.

 

5.     Select Yes when prompted to gain the latest version of NuGet.

 

6.     Selecting Yes to All when downloading from the PSGallery.

 

7.     After the script completes you will now see the Azure Automation ISE Add On as second tab.

 

8.     From the Configuration Tab of the Azure Automation ISE Add On, click the Browse button and select the C:\PS cmdlets directory.

 

9.     Next click the Sign In Button which will launch an Azure Sign In menu.  Use the account: autodeploy@<yourdirectory>.onmicrosoft.com with the password: demo@pass123.

 

10.  Once logged in the Window will fill in the details from Azure.

NOTE:  You may receive an error, but it shouldnt have any outcome to the use of the Account.

11.  Click the Runbooks Tab and see if they are now populated. 

 

12.  Click the SourceTest Runbook and then at the bottom press Test Draft in Azure and see how the script executes.

Summary

In this exercise, you configured the Authoring Tool for PowerShell ISE to connect to your Azure Automation Account from LABVM or your local machine. You also created an additional Assets that was used to login and execute script from your local machine right into Azure.
