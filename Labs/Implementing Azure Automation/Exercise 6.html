Exercise 6: Creating Webhook and Trigger a Runbook via a Webhook

Overview

In this exercise, you will create and configure a Webhook and trigger an action on an Azure Virtual Machine. The purpose of this Exercise is to use a Webhook for an automated action against an Azure Virtual Machine.

 

Time Estimate

30 minutes

 

Task 1: Creating a Webhook

1.     From the Lab-Deploy Automation Account select Runbooks, then Shutdown-VM and finally Webhooks.

 

2.     Select +Add Webhook,

 

3.     Then create new Webhook.

 

4.     Enter the name VM Shutdown and accept the default of Yes for Enabled.

 

5.     Set the expiration one hour from now and copy the URL.

 

Note: You will not gain another chance to copy the URL once the Webhook is created, ensure you have it saved in allocation for retrieval later.

 

6.     Once you have the URL copied select OK to create the Webhook.

 

7.     Click Parameters and run settings.

 

8.     Enter the values for the Parameters with VMNAME being a VM other than your LABVM and Resource Group Name where it resides.  In example case the VMNAME: WEBVMUS RG: OGDEMOTM.

 

9.     Select OK to confirm your settings then Create. Your Webhook is now enabled.

Task 2: Triggering the Webhook

1.     From the Azure Portal select Virtual Machines and then click on YOURVM.

NOTE: Start your VM if it is currently deallocated.

 

2.     Under monitoring select Alert Rules.

 

 

3.     Select Add metric alert at the top.

 

4.     Complete using the following inputs for the Add rule blade and then select OK:

a.     Name: Reserve Compute.

b.     Description: Shutdown VM to conserve cost on Compute.

c.      Metric: Percentage CPU

d.     Threshold: 50

e.     Period: Over the last 5 minutes

f.       Notify via: Check the box to notify via email and enter your email address so that you can be notified.

g.     Webhook:  Paste the Webhook URL into the Webhook field and select OK.

 

 

5.     Fire the alerts by connecting to YOURVM via Remote Desktop Connection from within your LABVM RPD Session.  Do this by connecting to the Azure portal from LABVM and pressing Connect YOURVM overview blade.

 

6.     Once connected to YOURVM copy the CPUSTRES.EXE application from your C:\OpsgilityTraining\Opsgility Automation Resources on your LABVM to the desktop of YOURVM.

 

7.     To trigger the Alert by double-clicking on the CPUSTRES application you just copied to the desktop of YOURVM.

 

8.     Select All Threads as Active and configure all Thread Priority to Highest and Activity to Busy.  The machine will push the machine to 100% CPU.

 

 

9.     Wait for the 5-minute threshold to elapse triggering the Webhook.

 

10.  From the Lab-Deploy Automation Account select Jobs to view the status of the Webhook once the alarm is received.

 

 

11.  Allow the process to run and confirm that your Virtual Machine is indeed Shutdown and deallocated.

 

12.  Finally ensure you receive both emails on the alert. One making you aware and the second with the automated action you configured for use.

 

Summary

In this exercise, you configured and triggered a Webhook to shut down a Virtual Machine to conserve computer cost on an Azure VM. From this Exercise, you have gained the knowledge on how to configure and deploy a webhook against your environment.
