Exercise 8: Working with Integration Modules

Overview

In this module, you will learn the how to deploy integration modules to your Azure Automation environments. You will also learn how to leverage modules and their function to apply configurations against resources in the cloud. You will also gain the ability on how to effectively use PowerShell package repositories such as PowerShellGallery.com to gain quick access to already created modules. A Linux command module will be used to automate the installation of an Apache Web Server.

 

Time Estimate

60 minutes

 

Task 1: Deploying Modules from PowerShell Gallery

 

1.     From LABVM navigate to Lab-Deployment and select Assets then finally Modules.

 

2.     Azure Automation has been updated with the PowerShell Gallery built-in.

Note: You can visit https://www.powershellgallery.com/ and download the modules here if you like

 

3.     Select Browse Gallery from the modules menu.

 

4.     In the search box type SSH and press enter.  This module will be used to manage the Linuxvm that is in the DSCRG resource group.

 

5.     Press Import and OK to confirm.

 

6.     Allow the process to complete as it can take a few minutes. Once done you will now see an activity Invoke-SSHCommand present. Also, take note that this module is not a Global Module. This indicates that the creator of it will not be updating it with another version on a feature rich basis.

 

7.     Close out of the Browse Gallery menu bringing you back to Module menu list. Notice SSH is now present at the bottom of the list of Modules.

 

8.     Close Modules and select Connections within the Assets menu.

 

9.     Select Add a Connection.

 

10.  In the Name field enter LinuxWebInstall and from the type menu select SSH. Enter the IP Address for the computer name, 22 for the port, demouser for the User Name and demo@pass123 for the password. Click Create to complete the configuration.

Note: To locate the Public IP Address you can view the DNS configuration on the Linuxvm Public IP Address in the DSGRG resource group. DO NOT USE THE DNS NAME AS THE SSH Script cant handle it.

 

11.  Move to Runbooks in Lab-Deploy and click+Add a runbook > Create a New runbook.

 

12.  Name it ApacheInstall and select PowerShell as the type and Press Create.

 

13.  On LABVM using PowerShell ISE Open C:\OpsgilityTraining\Opsgility Automation Resources\ApacheInstall.ps1

 

14.  In the script windows press Crtl+a which will highlight all the code and then click Copy.

 

15.  Paste the code from the ApacheInstall into the Runook code editor by pressing Crtl+v and clicking the Allow Access button on the Clipboard warning.

 

16.  Save the runbook and open the Test pane.

 

17.  In the CONNECTIONNAME parameter box type LinuxWebInstall and then press Start.

 

18.  The Runbook will execute in the Test Pane.  When the test of your Runbook is completed view the output and visit the Public IP address of LinuxVM machine to ensure that Apache Web Server was installed.

 

 

Summary

In this task, you deployed an SSH Module with the SSH command to automate the process of installing and configuring the Apache web server against your linuxvm virtual machine. You also gained the ability to create and leverage a connection as an asset with the integrated module.
