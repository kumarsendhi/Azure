Exercise 3: Deploying different Azure Backup Solutions

Overview

In this exercise, you will configure the following scenarios:

·         Using an agent based backup solution to protect files and folders

·         Deploying Azure Backup Server that can backup specific workloads

·         Backing up an Azure Virtual Machine

In the next exercise, you will use them for actual backup and recovery scenarios.

Time Estimate

45 minutes

 

Task 1: Deploying Azure Backup Agent-based solution

1.    Within the Azure portal, click Virtual Machines, and open the virtual machine AGENTBackupVM1. Click connect and login using the following credentials:

a.    demouser

b.    Demo@pass123

 

2.    From within the virtual machine, disable IE Enhanced Security mode.

 

 

 

3.    From the Azure Portal, browse to the Azure Recovery Vault you created earlier.

 

4.    From the Vault Settings blade, select Backup in the Getting Started Section. This will launch the Backup configuration wizard in the detailed blade section.

 

5.    Use the following parameters as configuration settings:

a.    Backup Goal / Where is your workload running: on-premises

b.    What do you want to backup: Files and Folders

 

6.    This will switch to the Prepare Infrastructure step, opening the blade where you can download the respective Azure Backup Agent.

 

7.    Choose Download Agent for Windows Server or Windows Client. Store the install file (MARSAgentInstaller.exe) to your local admin station.

 

8.    Choose Download Vault credentials to register the server to the vault. Store the vault credentials file to your local admin station.

 

9.    Complete the ‘Configuration for Backup’ by pressing the OK button twice from the Azure Portal.

 

10. Browse to the folder where you downloaded both files. Start the Azure Backup Agent installation by executing the MARSAgentInstaller.exe

 

11. This launches the Microsoft Azure Recovery Services Agent Setup Wizard

 

12. Accept the default values for Installation Folder and Cache Location. Press Next to continue.

 

13. Leave the Proxy Configuration as is, and press Next to continue.

 

14. We recommend to choose Use Microsoft Update; press Next to continue.

 

15. Confirm the installation by pressing the Install button. Wait for the installation to complete.

 

16. Press the Proceed to Registration button.

 

17. Browse to the download location where you downloaded the Vault Credentials file. Select this file. Press Next to continue.

18. To secure the backup data, encryption will be used. Therefore, you need to provide a Passphrase. This could be a self-chosen password of minimum 16 characters. You can also use the Generate Passphrase button to create a random password.

 

 

19. Press the Browse button to select the location where the passphrase file should be stored. (For the purpose and ease of this task, we recommend storing this file on the server’s desktop. In a production environment, this file would be stored in a safe® location.)

 

 

20. Press Finish to continue the Registration Server Wizard.

 

21. Wait for the Server Registration to complete successfully.

 

22. Press Close to complete the installation of the Microsoft Azure Recovery Services Agent. Leave the Launch Microsoft Azure Recovery Services Agent option activated.

23. The Microsoft Azure Backup application will open.

Note:  The actual configuration of a backup job will be executed in a later exercise.

a.    opsaaddemo\demouser

b.    Demo@pass123

 

2.    From within the virtual machine, disable IE Enhanced Security mode.

 

 

 

3.    Install the .NET 3.5 Framework (Azure Backup Server has a dependency), by launching a console (enter CMD in the run menu) and executing the following command:

DISM /Online /Enable-Feature /FeatureName:NetFx3 /All

 

4.    From within Administrative Tools, click Computer Management, and then click Disk Management. When prompted click OK to initialize the disk. Close Disk Management now and do not format the disk. This disk was attached to the virtual machine during the template deployment and will be used later by the Azure Backup Service for storage.

 

5.    Launch IE and navigate to the Azure Portal, click More Services, browse to the Azure Recovery Vault you created earlier.

 

6.    From the Vault Settings blade, select Backup in the Getting Started Section. This will launch the Backup configuration wizard in the detailed blade section.

 

7.    Use the following parameters as configuration settings:

a.    Backup Goal / Where is your workload running: on-premises

b.    What do you want to backup: <select any of the server workloads like Hyper-V, Exchange, SQL Server, SharePoint,…>

 

8.    This will launch the Prepare Infrastructure Instructions, from where you must download the Azure Backup Server files and registration credentials.

9.    Click the Download button on Option 2 and download the vault credentials file to your desktop. This file will be used to authenticate the Azure Backup Server install to your recovery vault.

 

10. In option 1, click the Download link next to “Install Microsoft Azure Backup Server”. This redirects you to the Microsoft website, from where you can download the Azure Backup files.

Task 3: Deploying IaaS VM based solution

1.    Within the Azure portal, click Virtual Machines, and open the virtual machine IAASBackupVM1. Click connect and login using the following credentials:

a.    demouser

b.    Demo@pass123

 

2.    From within the virtual machine, disable IE Enhanced Security mode.

 

 

 

3.    From the Azure Portal, browse to the Azure Recovery Vault you created earlier.

 

4.    From the Vault Settings blade, select Backup in the Getting Started Section. This will launch the Backup configuration wizard in the detailed blade section.

 

5.    Make the following selections:

a.    Backup Goal / Where is your workload running: Azure

b.    What do you want to backup: Virtual Machine

 

6.    In the Backup Policy section, accept the DefaultPolicy

7.    In the Items to backup, select IAASBackupVM1 from the list of Azure VMs.

8.    From the Azure Backup Vault settings in the Azure Portal, select Backup Items in the Protected Items section.

9.    Select the Azure Virtual Machine option in the details blade, which now has Backup Item Count set to 1.

10. Select Azure Virtual Machine, which will open up another blade, listing IAASBackupVM1 as Backup Item.

Note: you can safely ignore the Warning (initial backup) notification in the Last Backup Status column. This is by design for a new configured backup item.
