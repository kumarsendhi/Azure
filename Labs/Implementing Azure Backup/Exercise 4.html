Exercise 4: Configuring the Solutions

Overview

In this exercise, you will walk through configuring the different backup solutions.

Time Estimate

90 minutes

Task 1: Configure the Azure Backup Agent-based solution for protection and restore

In this task, you will configure a backup job, protecting several files and folders from the AgentBackupVM1 machine to Azure Vault, followed by deleting several of these files and folders from the server and performing a recovery task from Azure Vault and restoring this data.

1.    Log on to the AgentBackupVM1 machine using an RDP session. If the Azure Backup console is not opened anymore, launch it from the Start Menu.

 

2.    From within the Azure Backup / Action Pane, select Schedule Backup

3.    This starts the Schedule Backup Wizard.

4.    Press Next to continue.

 

5.    In the Select Items to Backup step, browse to the C:\Users\demouser folder and select it for backup.

6.    Press Next to Continue.

 

7.    This shows the Specify Backup Schedule configuration options. Leave the default value of 04:00 AM for now. Note there are 2 additional schedule timings available, allowing for up to 3 backup cycles per day.

 

8.    In the Select Retention Policy, accept the default values. Note the different retention settings available for Daily / Weekly / Monthly / Yearly Retention.

9.    In the Choose Initial Backup Type, select Automatically over the network. While not used for this exercise, notice the option for offline backup is also available, asking for several Azure-specific information related to your Azure Subscription in which the Azure Recovery Vault is configured.

10. Finish this Backup configuration wizard by confirming the last step. This creates the actual Backup job.

This completes the configuration of a scheduled backup job.

 

11. To complete this exercise and be able to restore data, we are not going to wait for the scheduled time, but run a manual backup job. To achieve this, navigate to the Action Pane, and select Backup Now

12. Since we only have 1 backup job configured on this server, the “backup now” action will recognize this job and execute it.

13. Wait for the backup to complete successfully.

In this last part of this task, you will delete data from the server, which will be recovered by performing a restore operation.

14. Browse to the folder you selected for backup (C:\users\demouser), browse to the Download folder and delete its content.

 

15. From the Azure Backup program, navigate to the Action Pane and select Recover Data.

16. From the Getting Started wizard, select the local server to restore data from.

17. In the Select Recovery Mode, select Individual Files and Folders.

 

18. In the select the volume and date, select the C:\ folder. Notice the calendar shows the dates in bold which had a backup job; notice the Time on which the backup job ran. Select this job and press the Mount button at the bottom of the window. This will download the backed up data volume from Azure, and opening it in the file explorer.

 

19. Select the files in the Download folder, and copy them over to the original location C:\users\demouser\Downloads.

 

20. From the Azure Backup home screen, notice the status of the Last Recovery changes to successful, showing date and time.

 

This completes the task in which you configured a backup job, as well as performed a recovery operation to restore data from the Azure Vault, on a system that was protected with the Azure Backup Agent-based solution.

 

Task 2: Configure Backup for an Active Directory Domain Controller.

In this task, you will configure the firewall on the Active Directory Domain Controller to allow remote access from the Azure Backup Server. You will then configure a backup job using the Microsoft Azure Backup Server (MABS) solution, protecting an Active Directory Domain Controller.

1.    Within the Azure portal, click Virtual Machines, and open the virtual machine MABSADDSVM1. Click connect and login using the following credentials:

a.    demouser

b.    Demo@pass123

 

2.    From within Server Manager, click Tools -> Windows Firewall with Advanced Security

 

3.    Right click on Inbound Rules, and click New Rule.

 

4.    Select Custom for the type of rule to create.

 

5.    Click Next on the next two screens to accept the default settings.

 

6.    Specify the IP address of the MABBSServerVM1 virtual machine to ensure the rules only apply to this VM.

 

7.    Accept the defaults for the rest of the settings and on the final screen name the rule AzureBackupServer.

 

8.    Within the Azure portal, click Virtual Machines, and open the virtual machine MABSServerVM1. Click connect and login using the following credentials:

a.    demouser

b.    Demo@pass123

 

9.    Once logged on to the server start the Azure Backup Server program. Click on Management, then Disks, and then Add.  

 

 

10. Add Disk 2 to the storage pool and click OK.

 

11. Read through the warning, and click Yes to confirm.

 

 

12. Click on Management, then Production Servers, then click the Add button.

 

 

13. This launches the Protection Agent Install Wizard. In the Select Agent Deployment Method, choose Install Agents.

 

14. In the list of computers shown, select MABSAADDSVM1, and add it to the Selected Computers list. Press Next to continue.

 

15. Enter the credentials which provide admin access to the selected computer

 

16. Proceed with the install, and allow the restart of the machine. The MABS agent will now get installed to the selected computer remotely.

 

Now the MABS Agent is installed to the remote machine, we can configure the actual backup protection.

 

17. From the MABS Console, click on Protection -> New, which will launch the Create New Protection Group wizard

18. In the Select Protection Group Type, select Servers

19. In the Select Group Members, browse to the MABSAADDSVM1 machine. By opening up the subfolders, notice it recognizes the System State, including Active Directory. Select this folder as backup source.

 

20. In the Select Data Protection method, verify both options short-term backup to DISK and I want Online Protection are flagged

 

21. In the Short Term Goal wizard step, accept the default of 5 days retention for this task.

22. In the Choose replica creation method, select Automatically over the network

 

23. Select to Run consistency check in the next step of the wizard

24. In the Specify Online Protection Data step, notice the details of the selected backup source

 

25. Accept the presented default for the Specify Online Backup Schedule

26. Accept the presented defaults for the Specify Online Retention Policy

 

27. In the Choose Online Replication, select Automatically over the network

28. This completes the steps in the Protection Group configuration wizard. Wait for the Protection Group to be created successfully:

 

This completes the task in which you deployed the Microsoft Azure Backup Server agent to a remote Windows Server, followed by configuring a new Protection Group for this selected server and configuring a backup job.

 

In this last part of this task, you will perform a restore of the System State / Active Directory Server component, by performing a restore operation.

29. From the MABS Server Console, go to Protection. Notice the Protection Group 1 which contains the Active Directory Domain Controller, is in a failed state for now, with an error message Replica is inconsistent.

 

Obviously, you need to troubleshoot what is going on and what the reason is for this failure.

30. From the MABS Server, open the Windows Event Viewer, by typing event viewer in the Start Screen.


31. From the Event Viewer, browse to Applications and Services Logs, and then browse to DPM Alerts. Notice the warning and error messages. Open the most recent error message.



32. Note the last sentence in the details of the error message, saying the Windows Server Backup (WBS) needs to be installed. Unfortunately, the explicit pointer to Windows 2008 is misleading, as Windows Server 2012 and 2012 R2 also need to have this component installed.

33. Within the Azure portal, click Virtual Machines, and open the virtual machine MABSADDSVM1. Click connect and login using the following credentials:

a.    demouser

b.    Demo@pass123

34. Start PowerShell from the Task bar or Start Menu.

35. Enter the following cmdlet: add-windowsfeature windows-server-backup

36. Once this feature is installed, close the PowerShell application, and log off from the virtual machine.

37. Reconnect to the MABSServerVM1 machine; From within the Azure Backup Server program, browse back to the Protection Group 1, and select the backup item Computer\System Protection\System State.

38. Right-click on the backup item, and choose Run Consistency Check from the context menu.

 

39. Read the warning message, and confirm to execute the consistency check by pressing Yes.


Note this process can take an extensive amount of time to complete. In our lab setup, it should take about 20min.

 

40. Once the consistency check is complete, the Protection Status of the backup item and Protection Group should be Green, and showing OK.



41. Assuming you accepted the defaults when configuring the Protection Group 1, there is no online protection yet, unless you are executing this lab after 9PM, which is the default time setting for online protection. However, we can simulate this by manually starting an online backup protection task.

42. Select the backup item again, right-click on it and select Create Recovery Point. In the popup window, select Online Protection in the “create recovery point for:” setting. Confirm with OK.



43. This opens another popup window, showing the progress of the data replication from the local disk to the online backup vault. Note the local disk backup is about 12GB in size. It takes about 15min before the online replication is completed.



44. Once the recovery point is created successfully, you can close this popup window.

45. At regular intervals, MABS will perform a sync from the on-disk protection to online protection. This is shown as “Synchronizing…” in the Protection Group and backup item Status column.

 

Also, notice the Online Protection has a status Enabled.

While you can wait for the Synchronizing to finish, to complete a full online recovery job, we can execute this from an on-disk protection as well. MABS still performs a first line of protection to a local disk in the backup server, which gets replicated to an online backup protection. The process for restoring however, is identical.

46. From the Azure Backup Server Console, select Recovery. This shows a calendar view with all current protection points for this server. Note it shows the latest recovery point available, and recovery is from Online.




47. From the top menu, select Recover. This starts the Recovery Wizard.



Note it shows the Computer\SystemProtection on MABSADDSVM1 as Recovery Source, as expected.

48. Press Next to continue.

 

49. In the Select Recovery Type, select Copy to a network folder. Press Next to continue.

50. In the Specify Destination step, click Browse. Browse to the following location:
opsaaddemo.local\MABSSERVERVM1\Shares\MTATempStore$

 

51. Notice this path gets completed in the destination option, and translated into the full Destination Path.

 

52. Press Next to continue.


53. In the Specify Recovery Options step, accept the default value of “Restore Security – Apply security settings of the destination computer”; press Next to continue.



54. Review your recovery settings in the Summary section, and confirm to execute the restore.



55. Wait for the restore to complete.

56. While the restore is ongoing, open the File explorer on the MABSServerVM1 machine. Browse to the restore destination shared folder selected earlier, which is C:\Program Files\Microsoft Azure Backup\DPM\DPM\Temp\MTA in this scenario.

 

57. From here, open the DPM_xx folder. This shows the Image disk, containing the System State backup image.

 

58. Head back to the Azure Backup Server console, and wait for the Restore job to complete. It shows a Recovery Status of successful.



59. This completes the restore operation of a System State backup using Microsoft Azure Backup Server.

Note: In a typical on-premises environment, the Active Directory Domain Controller should be rebooted in Active Directory Recovery Mode. Since in this lab you are using a Virtual Machine in Azure, this option is not available, as you can’t connect to the F8 boot option of the Azure Virtual Machine.

 

For being complete in our steps, here is a summarized view on what it takes to complete the restore of a system state backup on an on-premises physical or virtual machine:

-      Start the computer for which you want to restore the image to using the Windows DVD to match the system you are restoring, or boot the machine in Restore Mode or Active Directory Recovery Mode (depending if the restore is system state only, or Active Directory).

-      On the first screen verify language/locale settings and click Next.

-      On the Install screen comes up, select Repair your computer.

-      On the System Recovery Options page select Restore your computer using a system image that you created earlier > Next.

-      On the Select a system image backup screen select Select a system image > Advanced > Search for a system image on the network.

-      Select Yes if a warning appears.

-      Navigate to the share path, input the credentials, and select the recovery point.

-      This scans for specific backups available in that recovery point.

-      Select the recovery point and click Next.

-      In Choose how to restore the backup select Format and repartition disks. In the next screen verify settings and click Finish to begin the restore. Restart as required.

 

Task 3: Configure the Azure IAAS VM Backup solution for protection and restore of an Azure running Virtual Machine.

In this task, you will configure the Azure Backup option for IAAS running virtual machines to an already running Virtual Machine in Azure Resource Manager. Once the protection has been set up and ran successful, you will also perform a recovery operation of this Virtual Machine.

1.    From the Azure Portal, browse to the Azure Backup Vault you created earlier, and browse to Protected Items / Backup Items from the Vault settings.

 

2.    From the details blade that shows, select Azure Virtual Machine.

3.    Select the IAASBackupVM1 machine you configured earlier. This will open up the specific machine backup settings. Since this machine has been configured for backup, but no backup job has run before, it is in a Warning state.

 

4.    Press the Backup Now button, which will start a manual backup job for this VM.

5.    Specify a Retention time and confirm by pressing the Backup button at the bottom.

 

6.    From the Notification Area, wait for the message “Backup job successfully triggered”; select this message, which will redirect you to the Backup Job status view

7.    Select the running job for this VM, which will open a more detailed view in the next blade

8.    Wait for the backup job to complete.

This completes this task in which you configured and ran a backup job for an Azure Resource Manager running Virtual Machine.

 

In the next part of this exercise, you will perform a restore of this Azure IAAS virtual machine from Azure Backup.

9.    From the Azure Portal, browse to the OPSBackupRG Resource Group, and select the OpsBackupVault Recovery Services Vault.

10. From the OpsBackupVault blade that appears, browse to Backup Items under the Protected Items Section. Notice the Azure Virtual Machine – Backup Item count shows 1.

 

 

11. Select the item Azure Virtual Machine; this opens another blade, showing the Azure IAAS Virtual Machine IAASBackupVM1.

 

12. Select the IAASBackupM1 machine. This brings up another blade, showing the details of the last backup time, latest restore point,…

 

13. From the top menu, select Restore VM. This opens up the Restore blade, and moves to Step 1 – Select Restore Point.

 

14. From the list of restore points available, select the latest available. Confirm your selection by pressing the OK button.

15. This moves you to Step 2 – Restore Configuration. Complete the fields using below parameters:
a) Virtual Machine Name: RestoreVM1

b) Resource Group: OPSBackupRG

c) Virtual Network: OPSTrainingVNET

d) Subnet: Apps

e) Storage Account: <the storage account for your subscription>

 

 

 

16. Confirm the settings by pressing the OK button. This closes the Restore Configuration Blade.



17. Confirm the restore operation by pressing the Restore button. This brings up a notification “… Triggering restore for IAASBackupVM1” in the notification area.



18. Select this notification message. This will redirect you to the Backup Jobs blade, showing the details of all executed backup jobs, as well as the restore job in progress.



19. Wait for the Restore Job to complete successfully. For this specific Virtual Machine, it takes approximately 30 minutes.

20. Once the restore job has completed, move back to the Azure Resource Group OPSBackupRG in the Azure Portal. In the list of Azure Resources within this Resource Group, notice the RestoreVM1 Virtual Machine.



21. Select the RestoreVM1 Virtual Machine. From the details blade that opens, notice the Virtual Machine is running.

 

22. While the Virtual Machine is running, connecting to it using RDP is not possible by default, since there is no Public IP-address configured for this machine. This is as expected for a restored Virtual Machine. Remote Management is possible though, from any other machine in the same Subnet, for example the MABSADDSVM1 Domain Controller.

 

This completes this part of the lab in which you ran a restore job of an Azure IAAS Backup Virtual Machine.

Summary

In this lab, you deployed the different Azure Backup solutions available today, followed by configuring a backup job for each solution and executing a recovery operation for each solution.
