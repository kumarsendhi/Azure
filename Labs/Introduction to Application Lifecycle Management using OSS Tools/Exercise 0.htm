Task 1: Create a Chef trial subscription

Chef provides an automation system for building, deploying, and managing your infrastructure. Chef code is authored and managed using a folder structure called cookbooks (Chef specific). Each component of a cookbook is called a resource. Chef code contains reusable definitions that provide instructions for tasks such as configuring a web server. A cookbook may contain more than one recipe (more than one task to do), each recipe may contain reusable resources like files, templates, libraries and attributes etc.

Chef helps your write recipes (DSC code) and store them on Chef Servers (Either create your own Chef server by for example using an Azure Marketplace template or – like in this exercise – use a trial account for a hosted Chef Server environment).

In this exercise, you will learn how to create a Free Chef Server subscription (allows you to deploy/manage 5 Nodes (=Azure VMs in this scenario).

 

1.      Browse to https://manage.chef.io/signup, and complete the form. Note you have to use an existing email address, as you need to verify the account.

 

2.      Wait for the Chef Account validation email to arrive in your inbox

 

3.      Click on the verification URL in the email, and validate your account on the Chef portal.

 

4.      Create a new Chef Organization from the Chef Portal, using any name of your choice. The name must be unique to the Chef environment though.

 

Summary

This completes the creation and registration of a Chef trial account, in which you created a management organization. This environment will be used in the subsequent exercises.

Task 3: Deploy a Jenkins Server in Azure

Jenkins is an open source continuous integration tool written in Java. It provides continuous integration services for software development. It is a server-based system running in a servlet container such as Apache Tomcat.  In this exercise, you will deploy a Jenkins Server in Azure leveraging a prebuilt virtual machine image from the Azure marketplace.

1.      To deploy a Jenkins Server instance in Azure browse to https://portal.azure.com.  Click +New and type Jenkins in the search box and hit enter

 

 

2.      You will notice that there are numerous preconfigured Jenkins server available in the marketplace, for the purposes of the labs, choose the one submitted by Bitnami and click Create

 

3.      Specify the following on the Basics blade and click OK

·        Name: Jenkins

·        VM disk type: SSD

·        User name: demouser

·        Authentication type: password

·        Password: Demo@pass123

·        Resource group: JenkinsRG

·        Location: location nearest you

 

 

4.      Choose a size for your virtual machine (DS2_V2 Standard if available in your region) and click Select

 

5.      Leave the default settings and click OK on the Settings blade.

 

6.      Click OK on the Summary blade and Purchase on the Purchase blade to start the provisioning of your Jenkins server

 

7.      Wait for the Jenkins Virtual Machine to be deployed successfully. Once deployed, change the configuration of the Azure Public-IP, to have a fully qualified domain name (FQDN) for your Jenkins server by clicking Resource groups > JenkinsRG > jenkins (Public IP address)

8.      Click Configuration on the Settings tab of the Public IP address and specify a unique DNS name for your server and click Save

Before logging into the Jenkins portal, we will update the version of Jenkins to the latest as well as install other tools needed for our scenario.

9.      Connect to Jenkins via SSH

ssh demouser@jenkins101.centralus.cloudapp.azure.com

 

10.  Run the following command to elevate your user and update the packages of the OS

sudo su

apt-get update

 

11.  Run the following command to download the latest version of Jenkins

wget https://updates.jenkins-ci.org/latest/jenkins.war

 

12.  The following commands will clean up a folder and file before copying the latest version of jenkins.war to the directory it needs to reside in.

rm -r /opt/bitnami/apache-tomcat/webapps/jenkins

rm -r /opt/bitnami/apache-tomcat/webapps/jenkins.war

mv jenkins.war /opt/bitnami/apache-tomcat/webapps/

13.  Before installing the Azure Command Line tools, you need to install the node package manager by running the following command

apt-get install npm

 

14.  Type Y when you are prompted and hit Enter

The update of the components should take a few minutes to complete.

 

15.  After npm is installed, run the following commands to install and configure the Azure CLI

npm install azure-cli -g

sudo ln -s /usr/bin/nodejs /usr/bin/node

 

 

16.  Type azure to confirm that the Azure CLI was properly installed and choose whether you want to participate in data collection

 

 

17.  Download and Install the Chef DK by running the following commands (the Chef DK install does take several minutes to complete)

wget https://packages.chef.io/stable/ubuntu/12.04/chefdk_0.16.28-1_amd64.deb

sudo dpkg -i chefdk_0.16.28-1_amd64.deb

apt-get update

 

 

 

18.  Change directory so that you are at the root by running the following command

cd /

 

19.  You will now create a Chef Repository by running the following command

chef generate repo chef-repo
