Task 1: Create an Azure Storage Account

1.     Login to the Azure Management portal with your Azure credentials: http://portal.azure.com

 

2.     Click +New > Storage > Storage account

 

3.     Specify the following configuration for the Storage account and click Create.

·       Name: (unique name) blobstorage111

·       Deployment model: Resource manager

·       Account kind: General purpose

·       Performance: Standard

·       Replication: Read-access geo-redundant storage

·       Resource group: BlobStorageAppRG

·       Location: Location nearest you


4.     Once the deployment has completed, you will note the Access keys that will be used in a later exercise. Click on Resource groups, then BlobStorageAppRG, and the Storage Account.


5.     On the Storage Account blade, click on Access Keys under Settings


6.     The Access keys blade contains the information that you will need

  1. Lets set up a method that will set up our blob objects and create our container. We have to do this before we can access any of the blobs. The comments in the method explain whats going on.

 

One note about the container name. If you create a container and delete it and then try to create it again, you might hit the latency of removing a storage container  it wont let you create it again for a certain amount of time, like a few minutes. Rather than deal with this, were going to create a unique container name by taking our base name and concatenating it with the first 12 characters of a GUID. When you run this repeatedly before we add the cleanup code at the bottom, youll want to use a storage explorer application to remove all of the containers created.  Copy the code below under CloudBlockBlob cloudBlockBlob { get; set; }

public void SetUpObjects()

{

  //make the container name unique so if you run this over and over, you won't

  //  have any problems with the latency of deleting a container

  containerName = containerName + "-" + System.Guid.NewGuid().ToString().Substring(0, 12);

 

  //set the connection string

  ConnectionString =

    string.Format(@"DefaultEndpointsProtocol=https;AccountName={0};AccountKey={1}",

    storageAccountName, storageAccountKey);

 

  //get reference to storage account

  cloudStorageAccount = CloudStorageAccount.Parse(ConnectionString);

 

  //get reference to the cloud blob client

  //this is used to access the storage account blobs and containers

  cloudBlobClient = cloudStorageAccount.CreateCloudBlobClient();

 

  //get reference to the container

  cloudBlobContainer = cloudBlobClient.GetContainerReference(containerName);

           

  //create the container if it doesn't exist

  cloudBlobContainer.CreateIfNotExists();

       

  //set the permissions so the blobs are public

  BlobContainerPermissions permissions = new BlobContainerPermissions();

  permissions.PublicAccess = BlobContainerPublicAccessType.Blob;

  cloudBlobContainer.SetPermissions(permissions);           

}
