  1. Were going to want to get a list of the blobs. When you do this, it returns a list of the URI object. You can certainly display this, but its cluttered. Lets write a method that will parse out the actual blob name  stripping the blob endpoint and container off of the front of the URI. So if you have something like http://mystorage.blob.core.windows.net/mycontainer/mysubfolder/blobname.jpg, it will return mysubfolder/blobname.jpg. Add this method to your class. You can step through it if you want to figure out the math.

private string GetFileNameFromBlobURI(Uri theUri, string containerName)

{

    string theFile = theUri.ToString();

    int dirIndex = theFile.IndexOf(containerName);

    string oneFile = theFile.Substring(dirIndex + containerName.Length + 1,

        theFile.Length - (dirIndex + containerName.Length + 1));

    return oneFile;

}

 

  1. Now lets add a method that will display the list of blobs in the container. We can then run this to upload the files, and then call the method to list the blobs so we can see them. Were going to write a blank line at the beginning, and then a line that says begin list. Well write end list at the end. That way, if the list is empty, well be able to see that.

 

To get a list of blobs, you call a method on the container object. This is one of the hierarchy of objects that we populated in SetUpObjects. Each item in the returned list will have a Uri. Well call GetFileNameFromBlobURI to get just the blob name.

 

public void GetListOfBlobs()

{

    Console.WriteLine(string.Empty);

    Console.WriteLine("before list");

 

    foreach (IListBlobItem blobItem in

        cloudBlobContainer.ListBlobs(null, true, BlobListingDetails.None))

    {

        string oneFile = GetFileNameFromBlobURI(blobItem.Uri, containerName);

        Console.WriteLine("blob name = {0}", oneFile);

    }

    Console.WriteLine("after list");

 

}
