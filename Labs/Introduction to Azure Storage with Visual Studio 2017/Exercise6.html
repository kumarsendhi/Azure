Exercise 6: Blob properties and Metadata

In this exercise, you will learn how to read and write properties and meta data with Azure blobs.

Time Estimate

·       15 minutes

  1. Let’s start by adding an empty method called BlobProperties. Add a call to BlobProperties right above the console writes in BasicBlobOps.

 

  1. In BlobProperties, we’ll examine some properties and change the content type, and then modify the metadata for the blob. To examine or modify a blob’s properties, you first have to get a reference to the blob. Add this code to the BlobProperties method.

 

//let's look at snowy cabin, which is image4 and is in the pseudo-folder

cloudBlockBlob = cloudBlobContainer.GetBlockBlobReference(pseudoFolder + image4Name);

 

  1. Now let’s write a blank line to the console, and then display some of the blob properties. Once you have a reference to the blob, when you type in cloudBlockBlob and “.”, you will get IntelliSense showing the properties and methods available. Let’s print out the blob type, name, and URI.

 

Console.WriteLine(string.Empty);

 

//display some of the blob properties

Console.WriteLine("blob type = " + cloudBlockBlob.BlobType);

Console.WriteLine("blob name = " + cloudBlockBlob.Name);

Console.WriteLine("blob URI = " + cloudBlockBlob.Uri);

 

 

  1. To see the Properties properties (the system properties), you have to fetch the attributes of the blob first. Let’s do that, and then show the content type and size of the blob.

 

//update the system properties on the object and display a couple of them

cloudBlockBlob.FetchAttributes();

 

Console.WriteLine("content type = " + cloudBlockBlob.Properties.ContentType);

Console.WriteLine("size = " + cloudBlockBlob.Properties.Length);

 

  1. Now let’s change the content type. To do this, you just set it and then call SetProperties. After we do that, let’s fetch the attributes again and print it out again and when we run this, we’ll see the changed content type.

 

//change the content type from 'application/octet stream' to 'image/jpg'

cloudBlockBlob.Properties.ContentType = "image/jpg";

cloudBlockBlob.SetProperties();

 

//refresh the attributes and write out the content type again

cloudBlockBlob.FetchAttributes();

Console.WriteLine("content type = " + cloudBlockBlob.Properties.ContentType);

 

  1. If you run this now by hitting F5, you can see it print out this information about the properties and the new and old content type.

 

  1. Now let’s add metadata and then print it out. First, let’s write a small method to print out the metadata of the current blob. You just have to loop through the metadata and print each KeyValuePair. Don’t forget to fetch the attributes of the blob first, to populate the current values for the metadata. Here’s what PrintMetadata should look like:

 

public void PrintMetadata()

{

    //fetch the attributes of the blob to make sure they are current

    cloudBlockBlob.FetchAttributes();

    //if there is metaata, loop throught he dictionary and print it out

    int index = 0;

    if (cloudBlockBlob.Metadata.Count > 0)

    {

        IDictionary<string, string> metadata = cloudBlockBlob.Metadata;

        foreach (KeyValuePair<string, string> oneMetadata in metadata)

        {

            index++;

            Console.WriteLine("metadata {0} = {1}, {2}", index,

                oneMetadata.Key.ToString(), oneMetadata.Value.ToString());

        }

    }

    else

    {

        Console.WriteLine("No metadata found.");

    }

}

 

  1. Now let’s print the current metadata (currently blank). Then we’ll set the metadata and call SetMetadata to save the values and print out the metadata again. Add this code to the bottom of BlobProperties.

//print the current values

PrintMetadata();

 

//set some metadata and save it

cloudBlockBlob.Metadata["First"] = "number one";

cloudBlockBlob.Metadata["Second"] = "number two";

cloudBlockBlob.Metadata["Three"] = "number three";

cloudBlockBlob.SetMetadata();

 

//print it out again

PrintMetadata();

  1. Now let’s clear all of the metadata and print it out again – it should be a blank list.  Add this code to the bottom of BlobProperties .

//now clear the metadata, save the change,

//  and then print it again (empty list)

cloudBlockBlob.Metadata.Clear();

cloudBlockBlob.SetMetadata();           

PrintMetadata();

 

  1. Now when you run the console app by pressing F5, after it shows the properties, it will show (a) blank metadata, (b) metadata that we set, and (c) blank metadata again.

 

You can see all of the properties we printed, the change in the content type, and the first time, no metadata was found. The second time, it prints out what we set it to, and the last time after clearing the metadata, there’s no metadata found.
