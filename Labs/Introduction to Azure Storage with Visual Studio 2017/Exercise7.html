Exercise 7: Deleting Blobs

In this exercise, you will learn how to programmatically delete blobs from an Azure Storage Account.

Time Estimate

·       10 Minutes

 

  1. Lets see how to delete a blob from Azure, and then well start building what I call the cleanup routine. This is a routine that you run at the end of the console app that deletes everything youve created. This makes it cleaner to run the application repeatedly, and leaves no clutter behind.

 

To delete a blob, you have to get a reference to it, and then just call the delete method. Lets add a method called DeleteOneBlob and put in the code to delete the text file that we created in blob storage. This file was in the pseudo-folder, so we have to include that as part of the blob name.  Heres what the code looks like:

public void DeleteOneBlob()

{

    //delete one blob

    string blobName = image3Name;

    cloudBlockBlob =

        cloudBlobContainer.GetBlockBlobReference(pseudoFolder + textFileName);

    cloudBlockBlob.Delete();

}

 

  1. Add a call to DeleteOneBlob and GetListOfBlobs to BasicBlobOps, right before the console writes.

public void BasicBlobOps()

{

    //set up the objects

    SetUpObjects();

    UploadBlobs();

    GetListOfBlobs();

 

    DownloadBlobs();

 

    CopyBlob();

    GetListOfBlobs();

 

    BlobProperties();

  1. Now run the console app (by hitting F5) and it will show you in the output the listing after uploading the blobs, and then the listing after deleting one blob. You should be able to see that the text file is gone.

 

  1. Now lets put in a loop to delete all of the blobs in the container. We can take the loop we used for printing out the listing of blobs and just call delete on each one. Start by adding a method called DeleteAllBlobs. Heres what the code in that method will look like:

public void DeleteAllBlobs()

{

    List<string> listOBlobs = new List<string>();

    foreach (IListBlobItem blobItem in

        cloudBlobContainer.ListBlobs(null, true, BlobListingDetails.None))

    {

        string oneFile = GetFileNameFromBlobURI(blobItem.Uri, containerName);

        cloudBlockBlob = cloudBlobContainer.GetBlockBlobReference(oneFile);

        cloudBlockBlob.Delete();               

    }

}

 

  1. Lets add a method called CleanUp and have it call DeleteAllBlobs and then GetListOfBlobs. This will show you that there are no blobs left.

 

Lets also add the rest of the cleanup code. To leave no trace behind, we need to remove the download directory and the container in blob storage. To remove the container, you simply have to call the Delete method on the cloudBlobContainer object. And when we call the command to delete the local directory, lets check first to make sure we created it, and when we delete it, lets specify that it should delete recursively, so it will delete any subfolders and files. So heres what CleanUp looks like:

public void CleanUp()

{

    DeleteAllBlobs();

    GetListOfBlobs();

    cloudBlobContainer.Delete();

    if (!string.IsNullOrEmpty(localDownloadPath))

    {

        Directory.Delete(localDownloadPath, true);

    }

}

 

  1. Now add a call to CleanUp right before the console writes in BasicBlobOps.

public void BasicBlobOps()

{

    //set up the objects

    SetUpObjects();

    UploadBlobs();

    GetListOfBlobs();

 

    DownloadBlobs();

 

    CopyBlob();

    GetListOfBlobs();

 

    BlobProperties();

 

    DeleteOneBlob();

    GetListOfBlobs();

 

    CleanUp();

 

    Console.WriteLine("Press any key to exit.");

    Console.ReadLine();

}

 

  1. Now if you run this console application by pressing F5, it will show you the blobs uploaded, the blob properties, the blobs after deleting the text file, and the blobs after deleting all of them. (i.e. an empty list).

If you look in your upload folder, youll see the downloads folder is gone, and if you use a storage explorer to look in blob storage, youll see that the container we created and used is also gone.  

 

 

Summary

In this lab, you learned how to use C# to create a container and upload and download blobs between your local machine and Azure Blob Storage.
